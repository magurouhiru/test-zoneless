簡単なまとめ

  • 通常(zone あり)
    • click event
      • 全体に影響する。更新された値も表示される。
    • observable change
      • 全体に影響する。更新された値も表示される。
  • zoneless
    • click event
      • 全体に影響する。更新された値も表示される。
    • observable change
      • 全体に影響しない。signal を使用していれば更新された値が表示される。していれば表示に変化なし。
      • 内部の値は変わっているので、click event でDoCheck 以降がトリガーされれば、更新された値が表示される。
      • npm run start で実行時にはRuntimeError: NG0100 が表示される。

簡単な説明

コンポーネントは以下のような入れ子構造となっている。 
    check-event-observable.component
      ├─ no-signal-01.component
      │     ├─ no-signal-base.component
      │     └─ no-signal-02.component
      │           └─ no-signal-base.component
      └─ signal-01.component
            ├─ signal-base.component
            └─ signal-02.component
                  └─ signal-base.component
(no-)signal-base.component には以下の機能がある。
  • click event：押すとflag の真偽値が反転するボタンとflag の値表示。
  • Observable change：interval(1000).pipe(take(5)) の値表示。
  • ライフサイクルメソッドの発火回数表示。(console)
