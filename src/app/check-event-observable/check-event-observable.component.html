check-event-observable contents

簡単なまとめ

  • 通常(zone あり)
    • click event
      • 全体に影響する。更新された値も表示される。
    • observable change
      • 全体に影響する。更新された値も表示される。
  • zoneless
    • click event
      • 全体に影響する。更新された値も表示される。
    • observable change
      • 全体に影響しない。signal を使用していれば更新された値が表示される。していれば表示に変化なし。
      • 内部の値は変わっているので、click event でDoCheck 以降がトリガーされれば、更新された値が表示される。
      • npm run start で実行時にはRuntimeError: NG0100 が表示される。
