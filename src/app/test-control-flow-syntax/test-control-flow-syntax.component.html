簡単なまとめ

  • withZone
    • 全コンポーネントのngDoCheck()が発火し、更新された値も表示される。
  • Zoneless
    • 全コンポーネントのngDoCheck()が発火しない。
    • Signal を使用していれば、更新された値が表示される。
    • Signal を使用していなければ、更新された値が表示されない。

簡単な説明

ngDoCheck() の発火回数はコンソールに表示される。
(no-)signal.component には以下のプロパティがある。
  • value: interval(1000).pipe(take(5)) の値。
  • flag: interval(1000).pipe(take(5)) の値 % 2 === 0 の真偽値。
  • array: interval(1000).pipe(take(5)) の値をpush した値。
