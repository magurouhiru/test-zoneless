簡単なまとめ

  • withZone
    • 全コンポーネントのngDoCheck()が発火し、更新された値も表示される。
  • Zoneless
    • Signal を使用した子コンポーネントのngDoCheck()は発火する。それ以外のコンポーネントは発火しない。
    • Signal を使用していれば、更新された値が表示される。
    • Signal を使用していなければ、更新された値が表示されない。

簡単な説明

ngDoCheck() の発火回数はコンソールに表示される。
(no-)signal-parent.component には以下のプロパティがある。
  • value: interval(1000).pipe(take(5)) の値。
(no-)signal-child.component には以下のプロパティがある。
  • inputValue: 親コンポーネントから受け取った値。
  • inputValueRequired: 親コンポーネントから受け取った値。(必須)
